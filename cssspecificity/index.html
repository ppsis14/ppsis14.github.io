CSS Specificity

Specificity หรือการระบุอย่างเฉพาะเจาะจงในการจัดรูปแบบ ตกแต่ง element ของใน HTML เพื่อให้แสดงผลหรือแสดงรูปแบบออกมาตามที่ต้องการ ผ่านการเขียน CSS ไม่ว่าจะเป็นการกำหนด style ทั้ง 3 รูปแบบ คือ External Style Sheet, Internal Style Sheet และ Inline Styles

ปกติแล้วการใช้ Selector เพื่อเป็นการกำหนดค่าเพื่อใช้ในการแสดงผลของแต่ละ element เป็นสิ่งที่สำคัญและควรตระหนักถึง เพื่อการตอบสนองการใช้งานที่ดี ควรจัดลำดับความสำคัญในการใช้ CSS Selector ในการกำหนดการแสดงผล ซึ่งโดยปกติลำดับในการเขียนโค้ดจาก บน-ล่าง ก็ส่งผลต่อการใช้งาน CSS Selector ในการแสดงผลที่แตกต่างกัน แต่ความพิเศษกว่านั้นคือ CSS Selector ที่เลือกใช้แต่ละตัวจะมีค่าพลังของตัวเองที่แตกต่างกัน ซึ่งสามารถนำความสามารถเหล่านี้มาใช้ช่วยในการจัดลำดับความสำคัญในการใช้งาน CSS Selector ได้อีก

รูปภาพแสดงค่าพลัง (ค่า specificity) ของ CSS Selectorbr

จากรูปจะพบว่าแบ่ง Selector ออกเป็น 3 กลุ่ม ที่มีค่าพลังแตกต่างกัน ก็คือหากกำหนด style โดยใช้ Selector ที่เป็น ID จะส่งผลให้ค่าพลังเท่ากับ 100 หรือถ้าใช้ Selector ที่เป็น Class จะส่งผลให้ค่าพลังเท่ากับ 10 และหากใช้ Selector ที่เป็น element tag ทั้งหลาย จะส่งผลให้มีค่าพลังเท่ากับ 1 นั่นเอง

2.2 เมื่อมีการใช้ selector ตัวเดียวกัน ในการกำหนด style และพบว่ามีการเขียนลิ้งไปยังไฟล์ .css บริเวณ head tag จะพบว่า HTML จะแสดงผลออกมา ถ้า internal style เป็นอันล่างสุด จึงแสดงค่าที่กำหนดใน internal style แต่หากมีการกำหนด external style sheet เป็นอันล่างสุด ก็จะแสดงผลออกมาตาม external style sheet

3. กรณีแสดงผลของ element จากการกำหนด Inline Style ใน tag ที่ต้องการ

3.1 หากมีการกำหนดค่าใน selector ตัวเดียวกันกับ external style sheet แต่เนื่องจากการเขียนลิ้งไปยัง external style sheet ต้องกำหนดที่ head tag จึงทำให้ HTML แสดงผลออกมาตามที่ถูกกำหนดที่ inline style

3.2 หากมีการกำหนดค่าใน selector ตัวเดียวกันกับ internal style sheet ซึ่งต้องกำหนดที่ head tag เช่นเดียวกับ external style sheet จึงทำให้ HTML แสดงผลออกมาตามที่ถูกกำหนดที่ inline style

3.3 หากมีการกำหนดค่าใน selector ตัวเดียวกันกับใน external style sheet และ internal style sheet ที่ถูกกำหนดไว้ใน head tag ส่งผลให้ HTML แสดงผลออกมาตามที่ถูกกำหนดที่ inline style

จุดสังเกตเพิ่มเติม

จากที่กล่าวไปในข้างต้นว่า ในแต่ละ Selector ที่เลือกใช้งานจะมีค่าพลัง (ค่า specificity) ที่แตกต่างกันไปทำให้สามารถนำมาใช้ในการออกแบบ CSS ที่มีประสิทธิภาพในการเรียกใช้งาน Selector ที่มีลำดับความสำคัญที่แตกต่างกันได้ สามารถกำหนดค่าทับซ้อนกันได้

ตัวอย่างที่ 1


รูปภาพแสดงลำดับโค้ดของ tag p


รูปภาพแสดงผลการกำหนดค่าผ่าน p และเจาะจงมากขึ้นด้วยการใช้ div p


รูปภาพแสดงผลการกำหนดค่าผ่าน div p ที่มีความเจาะจงกว่า และตามด้วย p


รูปภาพแสดงผลที่สีของข้อความ


จากการคำนวณค่าพลัง (ค่า specificity) ของ element จากรูปภาพด้านบนอันแรก p {...} จะมีค่าพลังเท่ากับ 1 แต่ในถ้าคำนวณจาก div p {...} จะพบว่ามีค่าพลังคือ 1 + 1 = 2 จึงมีค่าพลังมากกว่า จึงทำให้ HTML แสดงผลสีของข้อความเป็นสีเหลืองดังรูปภาพข้างบน และถึงแม้จะมีสลับลำดับการกำหนดค่าเป็น div p {...} ก่อน แล้วตามด้วย p {...} การแสดงผลจะมีค่าเหมือนเดิมเพราะอย่างไรก็ตาม div p {...} มีค่าพลังมากกว่า

รูปภาพแสดงสีของข้อความ


จากการคำนวณค่าพลัง (ค่า specificity) ของ element จากรูปภาพด้านบน มีการใช้ .definition {...} ซึ่งเป็น class จะมีค่าพลังเท่ากับ 10 ในขณะที่ div p {...} และ p {...} มีค่าพลังเท่ากับ 2 และ 1 ตามลำดับ ดังนั้นการกำหนดค่าโดยใช้ .definition {...} จึงมีค่าพลังมากกว่า ทำให้ HTML แสดงผลให้สีข้อความเป็นสีดำดังภาพ

ตัวอย่างที่ 3



รูปภาพแสดงการกำหนดค่าของชื่อเจ้าของบทความโดยใช้ ID เป็น selector


จากการคำนวณค่าพลัง (ค่า specificity) ของ element จากรูปภาพด้านบน มีการใช้ #resource {...} ซึ่งเป็น ID จะมีค่าพลังเท่ากับ 100 จึงมีค่าพลังมากทำให้ HTML แสดงผลออกมีสีข้อความเป็นสีแดง


รูปภาพแสดงสีของข้อความ


แต่เมื่อมีการกำหนดค่าโดยการใช้ inlne style บริเวณชื่อของเจ้าของบทความ เนื่องจากการกำหนดค่าด้วย inline style มีค่าเท่ากับ 1000 ซึ่งกว่าทำให้ สีของข้อความที่เป็นชื่อเจ้าของบทความมีสีน้ำตาลตามการกำหนดด้วย inline style
